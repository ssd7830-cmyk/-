## 프롬프트

너는 웹 게임 프로토타입을 구현하는 시니어 프론트엔드 게임 개발자다.  
현재 작업 폴더는 `바탕화면/웹 게임/디펜스게임`이다.

이 프로젝트의 게임명은 `하미디펜스`다.

`하미디펜스`는 인생쌀집의 쌀알 마스코트 `하미`가 논밭과 쌀 창고를 지키는 브랜드형 라인 디펜스 웹 게임이다.  
`Plants vs. Zombies`의 구조적 재미인 라인 디펜스, 카드 배치, 자원 생산, 웨이브 방어는 참고하되, 캐릭터/배경/UI/적/연출은 완전히 다른 인생쌀집 세계관으로 구현해야 한다.

## 이번 작업 목표

이번에는 완성 게임을 만들지 말고 `Prototype 0.0.1`을 구현하라.

목표는 다음이다.

- 브라우저에서 바로 실행 가능한 웹 게임 프로토타입
- 5라인 x 9열 라인 디펜스 전장
- 카드 선택 후 칸 클릭으로 유닛 배치
- 쌀알 자원 시스템
- 기본 적 스폰과 이동
- 기본 공격, 투사체, 충돌, 피해 처리
- 방벽 접촉 전투
- 승리/패배 조건
- 하미 HUD와 말풍선

## 구현 기술

우선 순수 웹으로 구현하라.

사용 기술:

- HTML
- CSS
- JavaScript
- DOM + CSS Grid
- `requestAnimationFrame`

외부 라이브러리는 사용하지 마라.  
빌드 도구도 사용하지 마라.  
브라우저에서 `index.html`을 열면 바로 실행되어야 한다.

## Prototype 0.0.1 포함 범위

이번 버전에 반드시 포함할 것:

### 전장

- 5개 라인
- 9개 열
- 오른쪽에서 적 등장
- 왼쪽 끝에 적이 도달하면 패배
- 각 칸에는 방어 유닛 1개만 배치 가능

### 자원

자원 이름은 `쌀알`이다.

초기값:

- 시작 쌀알: 75
- 기본 자동 생산: 5초마다 25

자원은 화면 상단 HUD에 표시한다.

### 방어 유닛

이번 버전에는 아래 3개만 구현한다.

| 이름 | 역할 | 비용 | 체력 | 기능 |
|---|---|---:|---:|---|
| 퇴비통 | 생산 | 50 | 120 | 7초마다 쌀알 25 생산 |
| 새총 허수아비 | 공격 | 100 | 100 | 같은 라인의 가장 앞쪽 적에게 투사체 발사 |
| 호박 방벽 | 방어 | 50 | 400 | 적을 막고 시간을 벌어줌 |

카드 UI에는 각 유닛의 이름, 비용, 역할이 보여야 한다.

### 적

이번 버전에는 아래 2개만 구현한다.

| 이름 | 역할 | 체력 | 속도 | 공격력 | 특징 |
|---|---|---:|---:|---:|---|
| 진딧물 | 기본 잡몹 | 80 | 18 | 10 | 약하지만 자주 등장 |
| 애벌레 | 탱커 | 220 | 9 | 18 | 느리지만 체력이 높음 |

적은 오른쪽에서 등장해 같은 라인으로 왼쪽으로 이동한다.  
방어 유닛과 접촉하면 멈추고 해당 유닛을 공격한다.

### 공격

새총 허수아비는 같은 라인에 적이 있으면 자동으로 공격한다.

권장 수치:

- 공격력: 20
- 공격간격: 1.4초
- 투사체 속도: 초당 220px

투사체가 적과 닿으면 피해를 주고 사라진다.

### 웨이브

첫 스테이지는 약 3분짜리로 구성하라.

기획서의 `33. 첫 3분 스테이지 상세 시나리오`를 참고하되, 구현이 너무 길어지면 아래 간단 버전으로 구현해도 된다.

간단 웨이브:

- 20초: 진딧물 1마리
- 40초: 진딧물 2마리
- 65초: 애벌레 1마리
- 90초: 진딧물 3마리, 서로 다른 라인
- 120초: 애벌레 1마리 + 진딧물 2마리
- 160초: 마지막 웨이브, 진딧물 4마리 + 애벌레 2마리

모든 웨이브가 끝나고 화면에 적이 없으면 승리 처리한다.

### 하미 HUD

하미는 실제 이미지가 없으므로 CSS로 임시 표현한다.

하미 임시 디자인:

- 흰색/아이보리 쌀알 모양
- 작은 검은 눈
- 웃는 입
- 분홍 볼
- 짧은 팔 느낌

하미는 상단 왼쪽 HUD 또는 왼쪽 보호 구역에 보여야 한다.

하미 말풍선 상태:

- 시작: `쌀 창고를 지켜야 해요!`
- 적 등장 전: `저쪽에서 뭔가 와요!`
- 위험: `쌀 창고가 위험해요!`
- 승리: `해냈어요! 쌀 창고를 지켰어요!`
- 패배: `다음엔 더 잘 막을 수 있어요.`

## 시각 디자인 방향

절대 원작의 잔디밭 느낌을 그대로 따라 하지 마라.

원하는 느낌:

- 인생쌀집 앞 논밭
- 흙밭/밭고랑
- 쌀 창고
- 오른쪽 숲길 또는 풀숲에서 침입자 등장
- 따뜻하고 귀여운 농장 분위기

전장 방향:

- 왼쪽: 쌀 창고/하미 보호 구역 느낌
- 중앙: 5x9 밭고랑 격자
- 오른쪽: 적 등장 숲길/풀숲 느낌

색감:

- 흙색
- 벼색
- 아이보리
- 초록색은 보조로만 사용
- 원작처럼 밝은 잔디 초록이 화면 대부분을 차지하지 않게 할 것

## 타격감 필수 구현

Prototype 0.0.1에서는 최소한 아래 타격감을 구현하라.

- 적이 맞으면 120ms 정도 흔들리거나 깜빡임
- 적 체력이 줄어드는 것이 보이도록 작은 체력바 표시
- 적이 죽으면 그냥 사라지지 말고 작은 쌀가루/흙먼지 파티클 또는 fade-out 애니메이션
- 방벽이 맞을 때 살짝 흔들림

사운드는 이번 버전에서 없어도 된다.

## UI 요구사항

상단:

- 게임명 `하미디펜스`
- 하미 얼굴/캐릭터
- 쌀알 수치
- 방어 유닛 카드 3개

중앙:

- 5x9 전장
- 유닛, 적, 투사체 표시

하단 또는 좌측:

- 하미 말풍선
- 웨이브 진행 상태

상태 오버레이:

- 승리 화면
- 패배 화면
- 다시 시작 버튼

## 조작 방식

PC 기준:

1. 카드 클릭
2. 빈 칸 클릭
3. 비용이 충분하면 유닛 배치
4. 이미 유닛이 있는 칸에는 배치 불가
5. 비용이 부족하면 카드 또는 말풍선으로 부족함을 알려줌

모바일 드래그는 이번 버전에서 구현하지 않아도 된다.

필요한 상태:

- selectedCard

- projectiles

게임 루프:

1. 입력 처리
2. 자원/쿨타임 갱신
3. 웨이브 스폰
4. 방어 유닛 AI
5. 투사체 이동
6. 적 이동
7. 충돌/피해 처리
8. 사망/제거 처리
9. 승패 체크
10. 렌더링

## 구현 시 주의사항

- 기획서 MD 파일을 삭제하거나 크게 수정하지 마라.
- 원작 `Plants vs. Zombies` 캐릭터, 식물, 좀비, 카드 UI를 직접 모방하지 마라.
- 완성도를 욕심내기보다 플레이 가능한 작은 버전을 우선하라.
- 유닛과 적 수치는 반드시 한 곳에서 수정 가능하게 하라.
- CSS 클래스명과 JS 함수명은 읽기 쉽게 작성하라.
- 하드코딩을 최소화하되, 과한 추상화는 하지 마라.
- 주석은 복잡한 로직에만 짧게 달아라.

## 완료 기준

작업 완료 후 아래가 모두 가능해야 한다.

- `index.html`을 브라우저에서 열면 게임이 보인다.
- 카드 3개가 보인다.
- 쌀알 자원이 표시되고 시간이 지나며 증가한다.
- 퇴비통을 놓으면 추가 쌀알이 생산된다.
- 새총 허수아비를 놓으면 같은 라인의 적을 공격한다.
- 호박 방벽을 놓으면 적이 멈추고 공격한다.
- 진딧물과 애벌레가 웨이브 시간에 맞춰 등장한다.
- 투사체가 적에게 닿으면 피해를 준다.
- 적이 죽으면 연출 후 사라진다.
- 적이 왼쪽 끝에 도달하면 패배한다.
- 모든 웨이브를 막으면 승리한다.
- 승리/패배 화면에서 다시 시작할 수 있다.
